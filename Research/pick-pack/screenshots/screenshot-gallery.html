Workflow Overview

SOS Inventory uses a document-driven fulfillment model: Sales Order → Pick Ticket → Shipment. Pick tickets are optional warehouse instructions. Shipments remove items from inventory and generate packing slips. Integrates with ShipStation and UPS for label creation. Barcode scanning and mobile support available. Note: SOS help documentation is primarily text-based with minimal UI screenshots.

Key Metrics & Features

Pick Accuracy: 99.97%+ | Technology: CubiScan auto-boxing | Picking Model: Tote-based multi-order picking | Integration: Seller Central API

Packing Slip: Auto-generated at the pack station alongside the shipping label. System recommends box size using CubiScan dimensional data, packer scans items in, and label + packing slip print together. MCF orders support seller-branded or unbranded packing slips, configurable via API.

Amazon FBA - Main Program Amazon MCF - Multi-Channel Fulfillment

Why No Screenshots?

Amazon FBA's pick/pack operations happen entirely within Amazon's fulfillment centers using internal warehouse management systems. Sellers interact only through Seller Central (for inbound shipments and order tracking), not through the actual pick/pack workflow. The internal system uses scanner-driven tote-based picking, CubiScan auto-boxing, and automated carrier sorting — none of which have public-facing UI.
